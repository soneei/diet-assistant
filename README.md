# 饮食分析智能体

一个基于中医辨证 + 西医诊断交叉验证的饮食分析与食谱定制系统。

## 核心理念

**不是让你忌口，是让你在能吃的范围内吃得开心。**

## 产品架构

```
三层结构：
┌──────────────────────────────────────┐
│ 一层 · 诊断层                          │
│ 症状输入 → 中医辨证(11种体质) → 西医确诊交叉      │
├──────────────────────────────────────┤
│ 二层 · 食谱层                          │
│ 能吃vs不能吃 + 三餐建议 + 一周食谱 + 食疗方     │
├──────────────────────────────────────┤
│ 三层 · 快乐层（建设中）                    │
│ 做法多样化 + 安全调味 + 快乐触点 + 审美底线      │
└──────────────────────────────────────┘
```

## 核心交互（全部在对话中完成）

| 动作 | 用户说 | 智能体做 |
|------|--------|---------|
| 问 | "我能吃XXX吗？" | 双医交叉分析（西医+中医）→ 替代方案 → 报告卡 |
| 记 | "刚吃了XXX，感觉……" | 自动记入日志 → 更新食物敏感度画像 → 趋势总结 |
| 定 | "帮我安排明天的饮食" | 根据体质生成三餐+食谱卡 → 快乐烹饪说明 |

## 技术栈

- **AI引擎**：WorkBuddy对话系统 + 中医知识库（11种体质辨证）
- **数据层**：本地JSON文件（食物日志 + 敏感度画像）
- **输出端**：HTML报告卡（可直接在浏览器打开）

## 目录结构

```
diet-assistant/
├── SKILL.md                    ← 智能体行为规范（核心）

│   └── food_log.json           ← 饮食日志 + 敏感度画像
├── assets/

```

## 竞品差异

市面上的饮食类产品（Zoe、Noom、薄荷健康、汤头APP）都没有同时做到：
- 中医辨证 + 西医诊断交叉验证
- 有限食材内的好吃个性化
- 对话式的持续追踪
